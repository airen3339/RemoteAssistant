## 二、电脑端编译

### 2.1 开发模式运行

```bash
cd RemoteAssistant/desktop
npm install       # 首次需要安装依赖
npm start         # 启动开发模式
```

开发模式下直接运行 Electron，修改 `src/` 下的文件后重启即可看到变化。

### 2.2 打包构建

项目使用 **electron-builder** 进行打包。

#### 自动检测当前系统打包

```bash
npm run build
```

自动根据当前操作系统生成对应的安装包。

#### 指定目标平台打包

```bash
# Windows 安装包
npm run build:win

# macOS 安装包（需在 macOS 上执行）
npm run build:mac

# Linux 安装包
npm run build:linux
```

### 2.3 各平台输出产物

打包产物输出到 `desktop/dist/` 目录。

#### Windows

| 格式      | 文件                                      | 说明           |
| --------- | ----------------------------------------- | -------------- |
| NSIS 安装包 | `RemoteAssistant Setup x.x.x.exe`       | 标准安装程序    |
| 便携版     | `RemoteAssistant x.x.x.exe`              | 免安装直接运行  |

#### macOS

| 格式  | 文件                              | 说明          |
| ----- | --------------------------------- | ------------- |
| DMG   | `RemoteAssistant-x.x.x.dmg`     | 磁盘映像安装包 |

> **注意**：macOS 打包必须在 macOS 系统上执行。如需签名和公证，需要 Apple Developer 证书。

#### Linux

| 格式       | 文件                                   | 说明               |
| ---------- | -------------------------------------- | ------------------ |
| AppImage   | `RemoteAssistant-x.x.x.AppImage`     | 通用 Linux 可执行   |
| DEB        | `remote-assistant-desktop_x.x.x.deb` | Debian/Ubuntu 安装包 |

### 2.4 打包配置

打包配置位于 `desktop/package.json` 的 `"build"` 字段：

```json
{
  "build": {
    "appId": "com.remoteassistant.desktop",
    "productName": "RemoteAssistant",
    "directories": {
      "output": "dist"
    },
    "win": {
      "target": ["nsis", "portable"],
      "icon": "assets/icon.ico"
    },
    "mac": {
      "target": ["dmg"],
      "icon": "assets/icon.icns"
    },
    "linux": {
      "target": ["AppImage", "deb"],
      "icon": "assets/icon.png"
    }
  }
}
```

#### 自定义应用图标

在 `desktop/assets/` 目录下放置图标文件：

| 文件          | 格式     | 尺寸       | 用途    |
| ------------- | -------- | ---------- | ------- |
| icon.ico      | ICO      | 256×256    | Windows |
| icon.icns     | ICNS     | 1024×1024  | macOS   |
| icon.png      | PNG      | 512×512    | Linux   |

### 2.5 打包常见问题

| 问题                              | 解决方案                                    |
| --------------------------------- | ------------------------------------------- |
| Windows 打包缺少图标               | 在 `desktop/assets/` 下放置 `icon.ico`       |
| macOS 签名失败                     | 设置 `CSC_IDENTITY_AUTO_DISCOVERY=false` 跳过签名 |
| Linux AppImage 无法运行            | `chmod +x *.AppImage` 添加执行权限           |
| 打包时下载 electron 慢             | 设置镜像: `npm config set electron_mirror https://npmmirror.com/mirrors/electron/` |
| `electron-builder` 下载辅助工具慢  | 设置: `ELECTRON_BUILDER_BINARIES_MIRROR=https://npmmirror.com/mirrors/electron-builder-binaries/` |

## 三、手机端编译

### 3.1 项目初始化（首次）

Flutter 项目需要先生成平台目录才能编译：

```bash
cd RemoteAssistant/mobile

# 生成 android/ 和 ios/ 目录
flutter create --project-name remote_assistant_mobile .

# 安装 Dart 依赖
flutter pub get
```

### 3.2 开发模式运行

```bash
# 查看可用设备
flutter devices

# 运行到已连接的设备或模拟器
flutter run

# 指定设备运行
flutter run -d <device_id>

# 开启热重载的调试模式
flutter run --debug
```

### 3.3 Android 编译

#### Debug APK（调试版，用于开发测试）

```bash
flutter build apk --debug
```

输出：`build/app/outputs/flutter-apk/app-debug.apk`

#### Release APK（发布版）

```bash
flutter build apk --release
```

输出：`build/app/outputs/flutter-apk/app-release.apk`

#### 指定架构的 APK

```bash
# 仅 ARM64（主流手机）
flutter build apk --release --target-platform android-arm64

# 分架构打包（体积更小）
flutter build apk --release --split-per-abi
```

分架构打包输出：
- `app-arm64-v8a-release.apk` — ARM64 架构（推荐，覆盖大部分手机）
- `app-armeabi-v7a-release.apk` — ARM32 架构（老旧设备）
- `app-x86_64-release.apk` — x86_64 架构（模拟器/ChromeOS）

#### AAB 包（Google Play 上架）

```bash
flutter build appbundle --release
```

输出：`build/app/outputs/bundle/release/app-release.aab`

#### Android 签名配置

正式发布前需要配置签名。

1. 生成密钥：
   ```bash
   keytool -genkey -v -keystore ~/remote-assistant.jks -keyalg RSA -keysize 2048 -validity 10000 -alias remote-assistant
   ```

2. 创建 `mobile/android/key.properties`（**不要提交到版本库**）：
   ```properties
   storePassword=<你的密码>
   keyPassword=<你的密码>
   keyAlias=remote-assistant
   storeFile=<密钥文件绝对路径>
   ```

3. 修改 `mobile/android/app/build.gradle`，在 `android {}` 块中添加签名配置：
   ```groovy
   def keystoreProperties = new Properties()
   def keystorePropertiesFile = rootProject.file('key.properties')
   if (keystorePropertiesFile.exists()) {
       keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
   }

   android {
       signingConfigs {
           release {
               keyAlias keystoreProperties['keyAlias']
               keyPassword keystoreProperties['keyPassword']
               storeFile keystoreProperties['storeFile'] ? file(keystoreProperties['storeFile']) : null
               storePassword keystoreProperties['storePassword']
           }
       }
       buildTypes {
           release {
               signingConfig signingConfigs.release
           }
       }
   }
   ```

### 3.4 iOS 编译（仅 macOS）

#### 在模拟器上运行

```bash
# 打开 iOS 模拟器
open -a Simulator

# 运行
flutter run -d iPhone
```

#### 安装 iOS 依赖

```bash
cd mobile/ios
pod install
cd ..
```

#### Debug 构建

```bash
flutter build ios --debug --no-codesign
```

#### Release 构建

```bash
flutter build ios --release
```

#### 打包为 IPA（需要 Apple Developer 证书）

```bash
flutter build ipa --release
```

输出：`build/ios/ipa/remote_assistant_mobile.ipa`

#### Xcode 方式构建

也可以在 Xcode 中打开项目手动打包：

```bash
open mobile/ios/Runner.xcworkspace
```

在 Xcode 中：
1. 选择目标设备（真机或模拟器）
2. Product → Archive 生成归档
3. Distribute App 导出 IPA

#### iOS 签名配置

| 项目          | 说明                                      |
| ------------- | ----------------------------------------- |
| Apple ID      | 注册 Apple Developer Program ($99/年)     |
| Bundle ID     | 在 `ios/Runner.xcodeproj` 中设置，例如 `com.remoteassistant.mobile` |
| Provisioning Profile | 在 Apple Developer 后台创建       |
| 签名证书       | 在 Keychain Access 中管理                 |

### 3.5 HarmonyOS 编译

#### 方案一：Flutter 鸿蒙适配

```bash
# 确保已安装 Flutter 鸿蒙适配引擎
# 添加鸿蒙平台支持
flutter create --platforms ohos .

# 编译运行
flutter run -d ohos

# 打包 HAP
flutter build hap --release
```

#### 方案二：ArkTS 原生开发

1. 在 DevEco Studio 中打开 `mobile/harmony/` 项目
2. 点击 **Build → Build Hap(s)/APP(s)** 进行编译
3. 点击 **Run** 运行到鸿蒙设备或模拟器

## 五、CI/CD 参考脚本

### GitHub Actions 示例（电脑端 Windows 打包）

```yaml
name: Build Desktop (Windows)

on:
  push:
    tags: ['v*']

jobs:
  build-windows:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - name: Install dependencies
        working-directory: desktop
        run: npm ci
      - name: Build
        working-directory: desktop
        run: npm run build:win
      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: windows-build
          path: desktop/dist/*.exe
```

### GitHub Actions 示例（手机端 Android 打包）

```yaml
name: Build Mobile (Android)

on:
  push:
    tags: ['v*']

jobs:
  build-android:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-java@v4
        with:
          distribution: 'temurin'
          java-version: '17'
      - uses: subosito/flutter-action@v2
        with:
          flutter-version: '3.22.0'
      - name: Init project
        working-directory: mobile
        run: |
          flutter create --project-name remote_assistant_mobile .
          flutter pub get
      - name: Build APK
        working-directory: mobile
        run: flutter build apk --release --split-per-abi
      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: android-build
          path: mobile/build/app/outputs/flutter-apk/*-release.apk
```

## 七、编译速度优化建议

| 场景                    | 建议                                                |
| ----------------------- | --------------------------------------------------- |
| npm install 慢          | 使用淘宝镜像 `npm config set registry https://registry.npmmirror.com` |
| Electron 下载慢         | 设置 `electron_mirror` 为 npmmirror                  |
| flutter pub get 慢      | 配置 `PUB_HOSTED_URL` 和 `FLUTTER_STORAGE_BASE_URL`  |
| Gradle 下载慢           | 使用阿里云 Maven 镜像                                 |
| iOS pod install 慢      | 使用清华 CocoaPods 镜像源                             |
| 重复打包                | 电脑端使用 `npm run build` 增量构建                   |
| Android 多架构          | 使用 `--split-per-abi` 减小单个 APK 体积              |
