# RemoteAssistant 功能说明

> 版本：1.0.0  
> 更新日期：2025年

---

## 一、产品概述

RemoteAssistant（远程助手）是一套跨平台远程控制与AI助手管理工具，包含**电脑端**和**手机端**两个客户端。手机端通过局域网 WebSocket 向电脑端发送指令，电脑端接收并执行；同时电脑端集成了 OpenClaw 一键安装与主流大模型配置管理功能。

### 平台覆盖

| 端     | 技术栈    | 支持系统                      |
| ------ | --------- | ----------------------------- |
| 电脑端 | Electron  | Windows / macOS / Linux       |
| 手机端 | Flutter   | iOS / Android / HarmonyOS（鸿蒙） |

## 三、电脑端功能

### 3.1 仪表盘

| 功能              | 说明                                        |
| ----------------- | ------------------------------------------- |
| 连接状态显示       | 显示当前已连接的手机设备数量                   |
| 本机地址显示       | 自动检测本机所有IPv4网卡地址，方便手机连接      |
| WebSocket端口      | 默认 `9600`，可在设置中修改                   |
| OpenClaw状态       | 显示未安装 / 已安装 / 运行中                  |
| 最近指令           | 显示最近收到的手机端指令（最多5条）             |
| MCP Server        | 显示可用 Skills 数量和端口                     |
| HTTP API          | 显示 REST 端点数量和端口                       |

### 3.2 消息中心

- 实时接收手机端发来的所有消息和指令
- 区分**接收消息**（蓝色）和**发送消息**（绿色）
- 支持向所有已连接手机端回复消息
- 消息时间戳显示
- 自动滚动到最新消息

### 3.3 OpenClaw 管理

#### 一键安装

系统自动检测环境，选择最优安装方式：

| 检测条件          | 安装方式                                          |
| ---------------- | ------------------------------------------------- |
| 已安装 Docker     | 自动生成 `docker-compose.yml`，拉取镜像并启动容器   |
| 未安装 Docker     | 生成平台专属安装脚本（Windows `.bat` / Unix `.sh`） |

安装过程提供实时进度反馈（百分比 + 文字说明 + 日志输出）。

#### 运行管理

| 操作   | 说明                                      |
| ------ | ----------------------------------------- |
| 启动   | Docker 模式执行 `docker compose up -d`，直装模式执行 `npm start` |
| 停止   | Docker 模式执行 `docker compose down`，直装模式终止进程 |
| 打开界面 | 在浏览器中打开 `http://localhost:3210`    |

#### 配置参数

| 参数      | 默认值               | 说明                |

| 安装目录   | `~/.openclaw`        | 用户主目录下        |

### 3.6 命令执行安全机制

电脑端对手机发来的Shell命令实施严格的安全控制：

**允许执行的命令白名单：**

```
echo, date, whoami, hostname, ping, ipconfig, ifconfig,
systeminfo, dir, ls, pwd, cat, type, openclaw, ollama, docker
```

**禁止执行的危险命令：**

```
rm -rf, del /s/q, format, mkfs, dd if=, shutdown, reboot,
reg delete/add, net user, powershell -enc, curl|sh, wget|sh
```

命令执行超时时间为 **30秒**，输出截断为 **5000字符**。

## 五、通信协议

### 5.1 传输方式

| 参数    | 值                      |
| ------- | ----------------------- |
| 协议    | WebSocket               |
| 地址    | `ws://<电脑IP>:9600`    |
| 网络    | 局域网                   |
| 数据格式 | JSON                   |

### 5.2 消息格式

```json
{
  "type": "command | result | message | heartbeat | welcome",
  "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "timestamp": 1234567890000,
  "payload": {
    "type": "shell | openclaw | system | llm",
    "command": "...",
    "action": "...",
    "text": "..."
  }
}
```

### 5.3 消息类型

| type      | 方向        | 说明                          |
| --------- | ----------- | ----------------------------- |
| welcome   | 电脑→手机   | 连接成功后服务端发送欢迎消息    |
| heartbeat | 双向        | 30秒间隔心跳保活               |
| command   | 手机→电脑   | 发送指令请求                   |
| result    | 电脑→手机   | 指令执行结果返回               |
| message   | 双向        | 普通文本消息                   |

### 5.4 连接生命周期

```
手机端                              电脑端
  │                                   │
  │──── WebSocket Connect ──────────→ │
  │                                   │
  │←──── welcome {clientId} ─────────│
  │                                   │
  │──── heartbeat (每30秒) ────────→ │
  │←──── heartbeat ──────────────────│
  │                                   │
  │──── command {payload} ──────────→ │
  │                                   │ → 执行命令
  │←──── result {stdout/stderr} ─────│
  │                                   │
  │──── 断开连接 ───────────────────→ │
  │                                   │
  │     (5秒后自动重连)                │
  │──── WebSocket Connect ──────────→ │
```

## 六、MCP Server (Skills 服务)

### 6.1 概述

内置 MCP (Model Context Protocol) 服务，基于 SSE (Server-Sent Events) 传输协议，默认端口 `9601`。供 OpenClaw 或任何 MCP 兼容客户端连接并调用 Skills。

### 6.2 通信方式

| 参数    | 值                               |
| ------- | -------------------------------- |
| 协议    | HTTP + SSE                        |
| 默认端口 | `9601`                           |
| SSE 端点 | `GET /sse`                       |
| 消息端点 | `POST /messages`                 |
| 健康检查 | `GET /health`                    |
| 数据格式 | JSON-RPC 2.0                     |

### 6.3 可用 Skills（14个）

| 分类          | Skill 名称              | 说明                                          |
| ------------- | ----------------------- | --------------------------------------------- |
| 设备控制       | `send_to_mobile`        | 向已连接的手机发送消息或指令                     |
|               | `get_connected_devices` | 获取当前已连接的手机设备列表                     |
| 系统信息       | `get_system_info`       | 获取电脑系统信息（OS/CPU/内存/主机名等）         |
|               | `run_safe_command`      | 执行白名单内的安全命令                           |
| 大模型管理     | `list_llm_configs`      | 列出所有已配置的大模型                           |
|               | `call_llm`              | 调用指定大模型进行对话推理                       |
|               | `switch_llm`            | 启用或禁用指定模型配置                           |
| LLM-Studio    | `llm_studio_status`     | 查询本地 LLM-Studio 运行状态                    |
|               | `llm_studio_chat`       | 调用 LLM-Studio 对话接口                        |
| 文件操作       | `read_file`             | 读取电脑上指定路径的文件内容（安全路径限制）      |
|               | `list_directory`        | 列出指定目录的文件和子目录（安全路径限制）        |
| OpenClaw 管理  | `openclaw_status`       | 获取 OpenClaw 运行状态                          |
|               | `openclaw_start`        | 启动 OpenClaw 服务                              |
|               | `openclaw_stop`         | 停止 OpenClaw 服务                              |

### 6.4 OpenClaw 接入方式配置

在电脑端 OpenClaw 管理页面的「接入方式」卡片中，可选择 OpenClaw 与 RemoteAssistant 的通信方式：

| 模式                 | 默认 | 连接地址                           | 说明                              |
| -------------------- | ---- | -------------------------------------- | --------------------------------- |
| **MCP (SSE)**        | ?    | `http://localhost:9601/sse`            | 推荐。OpenClaw 自动发现并调用 Skills |
| HTTP REST API        |      | `http://localhost:9602/api`            | 通过标准 REST 接口调用             |

切换步骤：
1. 在 OpenClaw 管理页面的「接入方式」卡片中选择模式
2. 点击「保存接入方式」
3. 将显示的连接地址复制到 OpenClaw 对应配置中
4. 配置说明区域会自动展示对应模式的操作指引

## 七、HTTP REST API 服务

### 7.1 概述

内置 HTTP REST API 服务，默认端口 `9602`，供第三方脚本、Web 应用或自动化流程调用。与 MCP Server 共享相同的底层 Skill 能力，区别在于接口风格：REST API 提供标准的 HTTP 方法 + JSON 格式，更适合传统 Web 开发和脚本集成。

### 7.2 服务参数

| 参数       | 值                                |
| ---------- | --------------------------------- |
| 默认端口   | `9602`                            |
| Base URL   | `http://localhost:9602/api`       |
| 数据格式   | JSON                              |
| CORS       | 已启用（允许跨域）                 |
| 认证方式   | 可选 X-API-Key（在设置中配置）     |
| 请求体限制 | 1MB                               |

### 7.3 API 端点列表

| 方法   | 路径                    | 说明                                |
| ------ | ----------------------- | ----------------------------------- |
| GET    | `/api/health`           | 健康检查，返回服务状态               |
| GET    | `/api/tools`            | 列出所有可用工具和端点               |
| POST   | `/api/tools/:name`      | 通用工具调用（按工具名称）           |
| GET    | `/api/devices`          | 获取已连接的手机设备列表             |
| POST   | `/api/devices/send`     | 向手机发送消息或指令                 |
| GET    | `/api/system/info`      | 获取电脑系统信息                     |
| POST   | `/api/system/command`   | 执行白名单安全命令                   |
| GET    | `/api/llm/configs`      | 列出所有大模型配置                   |
| POST   | `/api/llm/chat`         | 调用大模型对话推理                   |
| PUT    | `/api/llm/configs/:id`  | 启用/禁用指定模型配置                |
| GET    | `/api/llm-studio/status`| 查询 LLM-Studio 运行状态            |
| POST   | `/api/llm-studio/chat`  | 调用 LLM-Studio 对话                |
| GET    | `/api/files`            | 列出目录内容（?path=...）            |
| GET    | `/api/files/read`       | 读取文件内容（?path=...）            |
| GET    | `/api/openclaw/status`  | 获取 OpenClaw 运行状态               |
| POST   | `/api/openclaw/start`   | 启动 OpenClaw                       |
| POST   | `/api/openclaw/stop`    | 停止 OpenClaw                       |

### 7.4 调用示例

#### 健康检查
```bash
curl http://localhost:9602/api/health
```

#### 获取系统信息
```bash
curl http://localhost:9602/api/system/info
```

#### 调用大模型
```bash
curl -X POST http://localhost:9602/api/llm/chat \
  -H "Content-Type: application/json" \
  -d '{"prompt": "你好", "max_tokens": 256}'
```

#### 通用 Skill 调用（按名称）
```bash
curl -X POST http://localhost:9602/api/tools/get_connected_devices \
  -H "Content-Type: application/json" \
  -d '{}'
```

#### 列出目录
```bash
curl "http://localhost:9602/api/files?path=/home/user/documents"
```

### 7.5 MCP vs HTTP API 对比

| 特性            | MCP Server (:9601)       | HTTP REST API (:9602)          |
| --------------- | ------------------------ | ------------------------------ |
| 协议            | SSE + JSON-RPC 2.0       | HTTP + JSON                    |
| 适用场景        | AI Agent / OpenClaw       | 脚本 / Web 应用 / 自动化       |
| 工具发现        | `tools/list` 方法         | `GET /api/tools`               |
| 工具调用        | `tools/call` 方法         | `POST /api/tools/:name` 或专用端点 |
| 实时推送        | ? SSE 流                 | ? 无（轮询）                   |
| 浏览器直接调用   | 需要 SSE 客户端           | ? fetch / axios / curl         |
