# RemoteAssistant 环境安装说明

> 版本：1.0.0  
> 更新日期：2025年

## 一、电脑端环境安装

### 1.1 Node.js 安装

电脑端基于 Electron 框架，需要 **Node.js >= 18**。

#### Windows

1. 访问 https://nodejs.org/ 下载 LTS 版本（推荐 18.x 或 20.x）
2. 运行安装程序，全部默认选项即可
3. 安装完成后验证：
   ```powershell
   node --version    # 应显示 v18.x.x 或更高
   npm --version     # 应显示 9.x.x 或更高
   ```

#### macOS

方式一：官网安装包
- 访问 https://nodejs.org/ 下载 `.pkg` 安装包

方式二：Homebrew（推荐）
```bash
brew install node@18
```

验证：
```bash
node --version
npm --version
```

#### Linux (Ubuntu/Debian)

```bash
# 添加 NodeSource 源
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证
node --version
npm --version
```

#### Linux (CentOS/RHEL)

```bash
curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo bash -
sudo yum install -y nodejs
```

### 1.2 Electron 项目依赖安装

进入电脑端项目目录，安装 npm 依赖：

```bash
cd RemoteAssistant/desktop
npm install
```

安装的核心依赖说明：

| 包名             | 版本     | 用途                     |
| ---------------- | -------- | ------------------------ |
| electron         | ^28.0.0  | 跨平台桌面应用框架        |
| electron-builder | ^24.9.0  | 安装包打包工具            |
| ws               | ^8.16.0  | WebSocket 服务端          |
| electron-store   | ^8.1.0   | 持久化配置存储            |
| uuid             | ^9.0.0   | UUID 生成                |

### 1.3 Docker 安装（可选，用于 OpenClaw）

如果需要通过 Docker 方式运行 OpenClaw，需先安装 Docker。

#### Windows

1. 下载 Docker Desktop：https://www.docker.com/products/docker-desktop/
2. 安装并启动 Docker Desktop
3. 验证：
   ```powershell
   docker --version
   docker compose version
   ```

#### macOS

```bash
# Homebrew 方式
brew install --cask docker

# 或从官网下载 Docker Desktop
```

#### Linux (Ubuntu)

```bash
# 安装 Docker
sudo apt-get update
sudo apt-get install -y docker.io docker-compose-plugin

# 启动并设置开机自启
sudo systemctl start docker
sudo systemctl enable docker

# 将当前用户加入 docker 组（免 sudo）
sudo usermod -aG docker $USER
newgrp docker

# 验证
docker --version
docker compose version
```

> **注意**：Docker 不是必须的。如果未安装 Docker，OpenClaw 管理器会自动切换为直接安装模式。

## 二、手机端环境安装

### 2.1 Flutter SDK 安装

手机端基于 Flutter 框架，需要 **Flutter >= 3.16**，**Dart >= 3.2**。

#### Windows

1. 下载 Flutter SDK：https://docs.flutter.dev/get-started/install/windows
2. 解压到目标目录，例如 `C:\flutter`
3. 将 `C:\flutter\bin` 添加到系统 PATH 环境变量
4. 验证：
   ```powershell
   flutter --version
   dart --version
   flutter doctor
   ```

#### macOS

```bash
# Homebrew 方式（推荐）
brew install flutter

# 或手动下载
# https://docs.flutter.dev/get-started/install/macos

# 验证
flutter --version
flutter doctor
```

#### Linux

```bash
# 下载 Flutter SDK
git clone https://github.com/flutter/flutter.git -b stable ~/flutter

# 添加到 PATH（写入 ~/.bashrc 或 ~/.zshrc）
export PATH="$HOME/flutter/bin:$PATH"

# 验证
flutter --version
flutter doctor
```

### 2.2 Android 开发环境

#### 安装 Android Studio

1. 下载地址：https://developer.android.com/studio
2. 安装 Android Studio 并运行
3. 通过 SDK Manager 安装：
   - **Android SDK** (API Level 33 或 34)
   - **Android SDK Command-line Tools**
   - **Android SDK Build-Tools**
   - **Android SDK Platform-Tools**

#### 配置环境变量

Windows:
```powershell
# 在系统环境变量中设置
ANDROID_HOME = C:\Users\<用户名>\AppData\Local\Android\Sdk
# 添加到 PATH
%ANDROID_HOME%\platform-tools
%ANDROID_HOME%\tools
```

macOS/Linux:
```bash
export ANDROID_HOME=$HOME/Android/Sdk     # 或 ~/Library/Android/sdk (macOS)
export PATH=$PATH:$ANDROID_HOME/platform-tools
export PATH=$PATH:$ANDROID_HOME/tools
```

#### 接受许可证

```bash
flutter doctor --android-licenses
```

#### 连接设备

- **真机**：手机开启 USB 调试，连接电脑
- **模拟器**：在 Android Studio 中创建 AVD 虚拟设备

### 2.3 iOS 开发环境（仅 macOS）

> iOS 开发只能在 macOS 上进行。

1. 安装 **Xcode**（App Store 下载，版本 >= 15）
2. 安装 Xcode 命令行工具：
   ```bash
   sudo xcode-select --install
   ```
3. 安装 CocoaPods：
   ```bash
   sudo gem install cocoapods
   # 或
   brew install cocoapods
   ```
4. 配置 Xcode：
   ```bash
   sudo xcodebuild -license accept
   ```
5. 打开 iOS 模拟器：
   ```bash
   open -a Simulator
   ```

### 2.4 HarmonyOS 开发环境（鸿蒙）

#### 方案一：Flutter 鸿蒙适配（推荐）

1. **安装 DevEco Studio** >= 4.0
   - 下载地址：https://developer.huawei.com/consumer/cn/deveco-studio/
   
2. **安装 HarmonyOS SDK**
   - 打开 DevEco Studio → Settings → SDK Manager
   - 选择并安装所需的 API 版本

3. **安装 Flutter 鸿蒙适配引擎**
   ```bash
   git clone https://gitee.com/openharmony-sig/flutter_flutter.git
   ```
   按照仓库说明配置环境变量。

#### 方案二：ArkTS 原生开发

1. 安装 DevEco Studio >= 4.0
2. 安装 HarmonyOS SDK
3. 创建 ArkTS 项目即可直接开发

### 2.6 Flutter 项目初始化

由于项目只提供了 `lib/` 源码和 `pubspec.yaml`，需要先生成平台脚手架文件：

```bash
cd RemoteAssistant/mobile

# 生成 Android/iOS 平台目录
flutter create --project-name remote_assistant_mobile .

# 安装依赖
flutter pub get
```

## 四、可能遇到的问题

### 4.1 npm install 失败

| 问题                          | 解决方案                                 |
| ----------------------------- | ---------------------------------------- |
| 网络超时                       | 设置淘宝镜像: `npm config set registry https://registry.npmmirror.com` |
| node-gyp 编译失败 (Windows)    | 安装 `npm install -g windows-build-tools` |
| 权限不足 (Linux/macOS)         | 不要使用 `sudo npm install`，修复 npm 权限 |

### 4.2 Electron 启动失败

| 问题                          | 解决方案                                 |
| ----------------------------- | ---------------------------------------- |
| GPU相关错误                    | 添加启动参数 `--disable-gpu`              |
| 沙箱错误 (Linux)               | `--no-sandbox` 或安装缺失的系统库         |

### 4.3 Flutter 问题

| 问题                          | 解决方案                                 |
| ----------------------------- | ---------------------------------------- |
| `flutter pub get` 超时        | 设置PUB镜像: `export PUB_HOSTED_URL=https://pub.flutter-io.cn` 和 `export FLUTTER_STORAGE_BASE_URL=https://storage.flutter-io.cn` |
| Android 许可证未接受            | 执行 `flutter doctor --android-licenses` |
| iOS 构建缺少 CocoaPods         | 执行 `sudo gem install cocoapods`         |
| Gradle 下载慢                  | 配置 Gradle 使用阿里云镜像                |

### 4.4 国内镜像配置汇总

```bash
# npm 镜像
npm config set registry https://registry.npmmirror.com

# Flutter/Dart 镜像（写入 .bashrc / .zshrc / PowerShell profile）
export PUB_HOSTED_URL=https://pub.flutter-io.cn
export FLUTTER_STORAGE_BASE_URL=https://storage.flutter-io.cn

# Gradle 镜像（android/build.gradle）
# 将 google() 和 mavenCentral() 替换为:
# maven { url 'https://maven.aliyun.com/repository/google' }
# maven { url 'https://maven.aliyun.com/repository/central' }
```

### 默认端口占用

确保以下端口未被其他程序占用：

| 端口   | 服务                   | 说明                                |
| ------ | ---------------------- | ----------------------------------- |
| `9600` | WebSocket Server       | 手机端连接通信                       |
| `9601` | MCP Server (SSE)       | OpenClaw / MCP Client 调用 Skills   |
| `9602` | HTTP REST API Server   | 第三方脚本 / Web 应用调用             |
| `3210` | OpenClaw Web UI        | OpenClaw 管理界面（安装后使用）       |

所有端口均可在电脑端设置页面中修改。
